Scenario 2: Position Calculation & Hierarchy Validation

What we're testing: The algorithm should validate that an element is actually a child of the parent before calculating its position.
Bug: Was returning position even for unrelated elements (tr[2] img treated as child of tr[3]).

Correct Case (Should Work)

Element: Row 3, Image 1
Parent: tr[data-row="3"]
XPath Match: ✅ Child of parent
Position: 1 (first child)
Expected Result: VALID

Incorrect Case (Should Fail)

Element: Row 2, Image 1 (UNRELATED)
Parent: tr[data-row="3"]
XPath Match: ❌ NOT a child of parent
Position: -1 (invalid)
Expected Result: INVALID
